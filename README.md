# Project_L.U.P.U.S
ローカルPC環境に常駐し、ユーザーのコンテキスト（タスク進捗、思考メモ、姿勢、コーディング規約）を監視・学習しながら、最適なLLMへ推論をルーティングするC++ベースの開発支援AIアーキテクチャを構築する。 本システムはプログラム本体（ロジック）と外部データ（プロファイル・プロンプト等）を完全に分離する「データ駆動設計」を採用し、高い拡張性と安全性を担保する。

プロジェクト名：L.U.P.U.S. (Local Utility & Productivity Unified System)
【概要】
ローカルPC環境に常駐し、開発者のタスク進捗、姿勢、コーディング規約を監視しながらAIによるメンタリングを行うC++ベースのシステム。単なるツールではなく、「どうすれば開発者が気持ちよく作業に集中できるか」というUXと、将来の拡張性を極限まで追求したアーキテクチャ設計を特徴としています。

【技術スタック】
コア言語: C++
設計パターン: State Machine（状態遷移）, Strategy（抽出抽象化）, Dependency Injection（依存性注入）
データ管理: JSON（システム設定・ルール）, Markdown（LLMプロンプト）

【コアアーキテクチャとアピールポイント】
1. 完全なデータ駆動設計（LogicとDataの分離）
ソースコード内に環境依存情報やプロンプトをハードコードすることを徹底的に排除しました。
システムプロンプト等の長文はエスケープ処理の不要な .md ファイルから読み込み、名前やAPIキーなどの短文設定は .json から読み込む構造を採用。プログラム（C++）は純粋なロジック処理に徹し、設定ファイルを差し替えるだけでシステムの振る舞いを完全に切り替えられる拡張性を実現しました。

2. セキュアな「同名ファイル・オーバーライド機構」
Git公開用の標準設定（config/）と、ローカル専用の機密・カスタム設定（private/）を分離し、起動時に「同名ファイルが存在すれば上書きマージする」ロジックをC++側で実装しました。
これにより、リポジトリの無菌化（トークン流出防止）と、ローカル環境での自由なペルソナカスタマイズを両立させています。

3. 疎結合を極めた Dependency Injection (DI)
各状態（State）クラスが外部ファイルや巨大な管理クラスに直接依存しないよう、DI（依存性の注入）を徹底しました。
StateMachine の初期化時に必要なプロンプトや名前の文字列のみを各 State へ注入する設計にすることで、各モジュールが独立したテストしやすい（責務汚染のない）純粋なコンポーネントとして機能します。

4. UXを追求した通知制御と静的解析
チーム開発におけるコード品質を担保するため、ファイル保存時に「真偽値メンバ変数の m_is プレフィックス漏れ」等を正規表現で即時検知する Linter モジュールを実装。
さらに、AIからの警告が無限ループしてユーザーの集中を削ぐ「警告疲れ（Alert Fatigue）」を防ぐため、キャッシュを用いた差分比較（デバウンス処理）を組み込み、本当に変化があった時のみ通知を行う実用的なUXを構築しました。
