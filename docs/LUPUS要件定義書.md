# プロジェクト要件定義書：L.U.P.U.S. (Local Utility & Productivity Unified System)

## 1. プロジェクト概要
ローカルPC環境に常駐し、ユーザーのコンテキスト（タスク進捗、思考メモ、姿勢、コーディング規約）を監視・学習しながら、最適なLLMへ推論をルーティングするC++ベースの開発支援AIアーキテクチャを構築する。
本システムはプログラム本体（ロジック）と外部データ（プロファイル・プロンプト等）を完全に分離する「データ駆動設計」を採用し、高い拡張性と安全性を担保する。

## 2. システムアーキテクチャと命名規則
* **コアロジック:** C++ 
* **命名規則の厳格化:** クラス名は`PascalCase`、真偽値メンバ変数は`m_is`プレフィックス、数値メンバ変数は`m_`プレフィックス + `camelCase` を必ず守ること。
* **推論ルーティングクラス:** 状況やAPIキーの有無に応じて、ローカルLLM（LM Studio API）とクラウドLLM（Claude/Gemini API）を自動フォールバックする。認証情報はハードコードせず、外部管理を阻害しないよう環境変数から読み込む設計とする。
* **音声パイプライン:** 入力はSilero VAD + Whisper.cppによる非同期処理。出力は外部TTS API（ローカルHTTPリクエスト）を叩くインターフェースを用意する。

## 3. セキュリティおよびプロファイル動的読み込み仕様
C++の初期化処理において、ソースコード内に特定の環境依存情報や機密性の高いプロンプトをハードコードすることを禁止する。以下のフォールバック付き上書きロジック（暗黙的カスタマイズ機構）を実装すること。

* **動作仕様:**
    1.  起動時に、Git管理対象外となるローカル固有のディレクトリ構造（`private/custom_profile.json`）の存在を走査する。
    2.  該当ファイルが存在する場合、それを最優先システムプロファイルとして動的に読み込む。
    3.  該当ファイルが存在しない場合は、Git管理対象である公開用の標準設定（`config/default_profile.json`）を読み込む。
* **期待される効果:**
    ソースコードの文字列を含め、リポジトリ全体を無菌化（汎用化）しつつ、ユーザーローカル環境のみで高度な個別ペルソナやカスタムプロンプトの上書き運用を可能にする。

## 4. コンテキスト監視モジュール
以下の外部環境を非同期で監視し、システムの状態フラグ（`m_isAtHome`, `m_isSlouching`等）をリアルタイム更新する。
* **Camera Vision:** OpenCV/MediaPipe等を用いた着座判定および姿勢（猫背）検知。
* **ネットワーク監視:** Windows APIによる特定のWi-Fi SSID接続判定。
* **タスク監視:** Notion APIによる未完了タスクの取得。
* **通知監視:** Windows Toast Notification APIをキャプチャし、Teams等の文脈を取得。
* **静的解析:** ユーザーが定めた上記のC++命名規則（`m_is`等）に対する違反を正規表現で検知。

## 5. プロファイル蓄積とスケジューラ
* **MemoryManager:** ObsidianのローカルMarkdownファイルやログをベクトル化して検索するRAGデータベースインターフェース。
* **MasterProfile:** C++の理解度や着座時間から算出される疲労度を保持する構造体。
* **スケジューラスレッド:** `std::chrono` 等を用い、設定時刻に自発的なシステムイベント（プッシュ通知等）を発火させる。

## 6. ステートマシン（状態遷移）
* **Standby:** 外出判定時。バックグラウンド待機。
* **TaskFocus:** 未完了タスクあり、コーディング規約違反検知、または姿勢悪化（`m_isSlouching == true`）時に遷移。ユーザーに警告・集中を促す。
* **TaskCompleted:** 自宅かつ全タスク完了時に遷移。リラックスモード。
